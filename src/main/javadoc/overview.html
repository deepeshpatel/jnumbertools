JNumberTools is the open source combinatorics and number-theory library. It provides 7 different types of permutation generation API's and 4 different types of combination generation API's

Permutation Generators
Permutation API Description
Unique Permutations All unique (n!) permutations of input list
Nth unique permutation Every Nth unique permutation in lex order starting from 0th permutation(input) and then generate every nth permutation in lexicographical order of indices of input values where each value is considered unique. Generating nth permutation is important because say, if we need to generate next 100 trillionth permutation of 100 items then it will take months to compute if we go sequentially and then skip the unwanted permutations because the total # of permutations is astronomical (100!= 9.3326E X 10157) This API will provide a mechanism to generate directly the next nth lexicographical permutation.
K-Permutation Unique permutations of size K. Permutations are generated in lex order of indices of input values, considering value at each indices as unique
Nth K-Permutation Generating every nth unique permutation of size k. Permutations are generated in lex order of indices of input values, considering value at each indices as unique.
Repetitive Permutation Utility to generate all permutations of given items and size where items can be repeated. Permutations are generated in lexicographical order of indices of input values, considering value at each indices as unique.
Nth Repetitive Permutation Utility to generate permutations with repeated values starting from 0th permutation(input) and then generate every nth permutation in lexicographical order of indices of input values. This is important because say, if we need to generate next 1 billionth permutation of [A,B,C,D,E,F,G,H,I,J,K,L,M] then it is not feasible to generate all 13^13=302875106592253 permutations and then skip billion permutations.
Repetitive permutation with limited supply repetitive permutations where every element in input has associated count which denotes home many times that element can be repeated. Permutations are generated in lex order of indices of input values, considering value at each indices as unique.


Combination Generators
Combination API Description
Unique Combinations All unique combinations (nCr) of input list
Nth Unique Combinations generating every nth unique combination of size k. Combinations are generated in lex order of indices of input values, considering value at each indices as unique.
This concept is important because count of combinations can grow astronomically large and to generate say, every 100 trillionth combination of 50 items out of 100 (100Choose50), we do not like to wait for 100's of hours to generate all 1.008913445 X 1029 combinations sequentially and then selecting the required one.


Subset Generators
Subset Type Description
All Subsets Generates all subsets of a given set.
Subsets in given size range Generates all subsets of a given set in a given size range
Generating unique permutations 
     JNumberTools.permutationsOf("A","B","C")
                 .unique()
                 .forEach(System.out::println);

 will generate following (all possible unique permutations of A,B and C in lex order) -
 [A, B, C]
 [A, C, B]
 [B, A, C]
 [B, C, A]
 [C, A, B]
 [C, B, A]
Generating Nth unique permutations 
         JNumberTools.permutationsOf("A","B","C")
                 .uniqueNth(2)
                 .forEach(System.out::println);

  will generate following (0th, 2nd and 4th) unique permutations of A,B and C in lex order)-

 [A, B, C]
 [B, A, C]
 [C, A, B]
Generating K-permutations 
  JNumberTools.permutationsOf("A","B","C")
      .k(2)
      .forEach(System.out::println);

 will generate following (all possible unique permutations of size(K) 2)-
 [A, B]
 [B, A]
 [A, C]
 [C, A]
 [B, C]
 [C, B]
Generating Nth K-Permutation 
  JNumberTools.permutationsOf("A","B","C")
      .kNth(2,2) //size =2 and skipTo every 2nd permutation starting from 0th
      .forEach(System.out::println);

 will generate following (0th, 2nd and 4th) K-permutation of size(K)=2 -
 [A, B]
 [A, C]
 [B, C]
Generating repetitive permutations 
      JNumberTools.permutationsOf("A","B","C")
            .repetitive(2)
            .forEach(System.out::println);

 will generate following (all possible repetitive permutations of size 2) -
 [A, A]
 [A, B]
 [A, C]
 [B, A]
 [B, B]
 [B, C]
 [C, A]
 [C, B]
 [C, C]
Generating Nth repetitive permutation 
      JNumberTools.permutationsOf("A","B")
                 .repetitiveNth(3,2) //size=3, repeat=2
                 .forEach(System.out::println);

 will generate following (0th, 2nd 4th and 6th) repetitive permutation of "A" and "B" of size 3 -

 [A, A, A]
 [A, B, A]
 [B, A, A]
 [B, B, A]
Generating repetitive permutations with limited supply 
     //here A can be repeated max 2 times and B can not be repeated(1 time)
     JNumberTools.permutationsOf("A","B")
                 .repetitiveWithSupply(2,1)
                 .forEach(System.out::println);

 will generate following -
 [A, A, B]
 [A, B, A]
 [B, A, A]
Generating unique combinations 
     JNumberTools.combinationsOf("A","B","C")
        .unique(2)
        .forEach(System.out::println);

 will generate following (all unique combinations of size 2 of A and B) -
 [A, B]
 [A, C]
 [B, C]
Generating Nth Unique Combination 
     JNumberTools.combinationsOf("A","B","C","D")
        .uniqueNth(2,1)
        .forEach(System.out::println);

 will generate following(0th, 2nd and 4th) unique combinations of all 6 possible combinations:
 [A, B]
 [A, D]
 [B, D]
Generating All subsets 
     JNumberTools.subsetsOf("A","B","C")
        .all()
        .forEach(System.out::println);

 will generate following -
    []
    [A]
    [B]
    [C]
    [A, B]
    [A, C]
    [B, C]
    [A, B, C]
Generating subsets in a given range 
     JNumberTools.subsetsOf("A","B","C")
        .inRange(2,3)
        .forEach(System.out::println);

 will generate following -

    [A]
    [B]
    [C]
    [A, B]
    [A, C]
    [B, C]
    [A, B, C]
